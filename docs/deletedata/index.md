### Notes

> NOTE: The `DeleteData` function is designed for use in landing pages, microsites, and SMS messages in Mobile Connect. For deleting rows from a data extension within an email, the `DeleteDE()` function should be used instead.

### Example 1: Basic Use

This example demonstrates how to delete a single row from a data extension named 'Subscribers' where the 'EmailAddress' column matches a specific value.

```html
%%[
/* Define the Data Extension name and the lookup values */
SET @deName = "Subscribers"
SET @emailToDelete = "example@example.com"

/* Check if the email address is not empty before attempting to delete */
IF NOT EMPTY(@emailToDelete) THEN
    /* Attempt to delete the data and store the number of rows affected */
    SET @rowsDeleted = DeleteData(
        @deName, /* Data Extension Name */
        "EmailAddress", /* Column Name to match */
        @emailToDelete /* Value to match in the Column */
    )

    /* Output the result of the deletion */
    IF @rowsDeleted > 0 THEN
        Output(Concat("Successfully deleted ", @rowsDeleted, " row(s) for email: ", @emailToDelete, "."))
    ELSE
        Output(Concat("No rows deleted for email: ", @emailToDelete, ". It might not exist or an error occurred."))
    ENDIF
ELSE
    Output("Error: Email address to delete cannot be empty.")
ENDIF
]%%
```

### Example 2: Advanced Scenario

This example shows how to delete multiple rows from a 'ProductOrders' data extension based on two criteria: 'OrderStatus' being 'Cancelled' and 'OrderDate' being before a specific date. This demonstrates deleting based on multiple column-value pairs.

```html
%%[
/* Define the Data Extension name and the lookup values */
SET @deName = "ProductOrders"
SET @statusToDelete = "Cancelled"
SET @dateThreshold = "2023-01-01"

/* Check if the status and date threshold are not empty before attempting to delete */
IF NOT EMPTY(@statusToDelete) AND NOT EMPTY(@dateThreshold) THEN
    /* Attempt to delete the data based on multiple criteria */
    SET @rowsDeleted = DeleteData(
        @deName, /* Data Extension Name */
        "OrderStatus", /* First Column Name to match */
        @statusToDelete, /* Value for the first column */
        "OrderDate", /* Second Column Name to match */
        @dateThreshold /* Value for the second column */
    )

    /* Output the result of the deletion */
    IF @rowsDeleted > 0 THEN
        Output(Concat("Successfully deleted ", @rowsDeleted, " cancelled order(s) before ", @dateThreshold, "."))
    ELSE
        Output(Concat("No cancelled orders found before ", @dateThreshold, " or an error occurred."))
    ENDIF
ELSE
    Output("Error: Order status or date threshold cannot be empty.")
ENDIF
]%%
```
