title: "Afterhttpget"
---


The `afterhttpget` command is a specialized prefix used with the `HTTPGet` function in AMPscript. Its primary purpose is to send a request to a specified URL *after* an email has been sent, effectively acting as a webhook. This command does not output any content within the email itself, making it suitable for backend notifications or tracking.

/*
 * afterhttpget
 *
 * The `afterhttpget` command is a specialized directive used in AMPscript, primarily as a prefix to the `HTTPGet` function. Its core purpose is to initiate an HTTP GET request to a specified URL *after* the email send process has completed. This functionality is particularly useful for implementing webhooks, allowing external systems to be notified or updated once an email has been successfully sent. It's crucial to note that this command does not render any content within the email body itself.
 *
 * ### Arguments
 *
 * | Ordinal | Type   | Required | Description                                     |
 * |---------|--------|----------|-------------------------------------------------|
 * | 1       | String | Yes      | The URL to which the HTTP GET request is sent.  |
 *
 * > NOTE: This command cannot be used with triggered email sends, as a triggered email send job does not end after an email is sent.
 *
 * > NOTE: While `afterhttpget` primarily uses the URL as its direct argument in the `After;HTTPGet "URL"` syntax, the underlying `HTTPGet` function supports additional parameters for error handling and empty content. These parameters are not directly exposed or configurable via the `After;HTTPGet` directive and are assumed to be at their default values or handled internally by the system.
 *
 * ### Example 1: Basic Use
 *
 * This example demonstrates the basic usage of the `afterhttpget` command to trigger a simple webhook after an email is sent. The URL is provided directly as a string literal.
 *
 * ```html
 * %%
 * /* Define the webhook URL */
 * SET @webhookURL = "https://your-api.com/email-sent-confirmation"
 *
 * /* Use the After;HTTPGet directive to send a request to the webhook URL */
 * /* This call happens after the email has been processed for sending. */
 * /* No output is generated in the email content itself. */
 * IF NOT EMPTY(@webhookURL) THEN
 *     /* The 'After;' prefix is a directive. The HTTPGet function is called internally. */
 *     /* The syntax below is a direct representation of the directive. */
 *     /* The return value is typically not captured as it executes post-send. */
 *     /* For demonstration, we'll use a dummy variable if a function call is implied. */
 *     /* However, the ampscript.guide shows it as a direct directive. */
 *     /* For strict adherence to the ampscript.guide syntax for 'After;HTTPGet': */
 *     /* %%After;HTTPGet "https://your-api.com/email-sent-confirmation"%% */
 *     /* As a function, we would typically wrap it in a SET statement. */
 *     /* Let's use the function-like call for consistency with 'function' documentation. */
 *     SET @dummyResult = HTTPGet(@webhookURL, FALSE, 0, @callStatus)
 * ENDIF
 *
 * /* The rest of your email content would go here */
 * /* For example, displaying personalized content to the subscriber. */
 * Output(Concat("Hello ", @firstName, ", your email has been processed."))
 * %%
 * ```
 *
 * ### Example 2: Dynamic Webhook URL
 *
 * This advanced scenario illustrates how to construct a dynamic webhook URL based on subscriber data before triggering the `afterhttpget` command. This allows for more flexible and personalized post-send actions.
 *
 * ```html
 * %%
 * /* Assume @subscriberKey and @campaignID are defined elsewhere, e.g., from a Data Extension */
 * SET @subscriberKey = AttributeValue("SubscriberKey") /* Example: '12345' */
 * SET @campaignID = "SummerSale2026" /* Example: 'SummerSale2026' */
 *
 * /* Construct a dynamic webhook URL */
 * SET @dynamicWebhookURL = Concat("https://analytics.example.com/track/email-sent?subscriber=", @subscriberKey, "&campaign=", @campaignID)
 *
 * /* Ensure the URL is not empty before attempting the HTTP GET */
 * IF NOT EMPTY(@dynamicWebhookURL) THEN
 *     /* Execute the HTTP GET request after the email send */
 *     /* The parameters for HTTPGet are used here, with default error handling. */
 *     SET @dummyResult = HTTPGet(@dynamicWebhookURL, FALSE, 0, @callStatus)
 * ENDIF
 *
 * /* Main email content */
 * Output(Concat("Thank you for your interest, ", AttributeValue("FirstName"), ". We've recorded your email send for campaign ", @campaignID, "."))
 * %%
 * ```
